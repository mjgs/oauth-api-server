Unified JWT Authentication Demo

Explore different authentication and authorization flows with JWTs, backed by a Node.js Express API.

1. Mock User Login (for PAT Management)

Use this to simulate logging in as a user. This grants you a short-lived user token which is required to generate Personal Access Tokens (PATs) on the PAT Management page.

  • testuser / password123 (User Role)
  • adminuser / adminpass (Admin Role)

Your User Token (short-lived): Copied!

2. Access API Endpoints Directly

Test protected API endpoints by manually pasting a JWT into the field below. You can get a user token from the "Mock Login" above, or obtain other tokens (PAT, API Key) via Postman/cURL using the `/oauth/token` or `/api/pat/generate` endpoints.

3. OAuth 2.0 Web Client Demo (External App)

This API acts as an Authorization Server. To see the full OAuth flow (Authorization Code Grant with PKCE) in action, you'll need to run a **separate web client application**. This client will redirect to this API's /oauth/authorize endpoint. Ensure the client's redirect_uri is correctly registered in this API's server.js file.

A reference client implementation can be found here:

4. Personal Access Token (PAT) Management

Navigate to the dedicated PAT management page to generate and revoke user-specific tokens. You must perform the mock login above first to be 'authenticated' for this page.

Go to PAT Management Page

5. Admin Panel

Access the administrative panel. This endpoint requires an 'admin' role, which you can obtain by logging in with adminuser/adminpass and then using its generated token for the protected API call above.
